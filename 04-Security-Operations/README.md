### Security Incident Reporting Procedure

Developed a formal Security Incident Reporting Procedure to establish clear processes for identifying, reporting, documenting, and escalating cybersecurity incidents across the organization.

The procedure defines employee responsibilities, reporting timelines, escalation paths, communication requirements, and documentation standards to ensure security events are addressed consistently and efficiently.

This document supports the organization's broader Incident Response Program and helps improve security visibility, response readiness, and compliance with governance requirements.

Frameworks Referenced:

NIST SP 800-61 Incident Response Guide
NIST Cybersecurity Framework (CSF)
CIS Controls
ISO 27001
