### RBAC Access Control Policy

Developed a Role-Based Access Control (RBAC) policy to establish standardized access management procedures across all departments within IsoTech Ink.

The policy defines user access responsibilities, approval workflows, privilege management standards, separation of duties requirements, and periodic access review processes to reduce the risk of unauthorized access and data exposure.

The policy was designed to address identified access control weaknesses and improve governance over user permissions, remote access, and sensitive business information.

Key Areas Covered:

• Role-Based Access Control (RBAC)
• Least Privilege Principles
• User Provisioning & Deprovisioning
• Access Approval Workflows
• Privileged Account Management
• Remote Access Controls
• Periodic Access Reviews
• Separation of Duties

Frameworks Referenced:

• NIST SP 800-53
• CIS Critical Security Controls
• ISO 27001 Access Management Controls

### Access Review Matrix

Developed a formal Access Review Matrix to support ongoing governance of user permissions across IsoTech Ink's business systems and applications.

The matrix establishes ownership, review frequency, approval requirements, and validation procedures for user access across departments, ensuring that access privileges remain aligned with business responsibilities and least privilege principles.

The document supports audit readiness, regulatory compliance, and identity governance by providing a repeatable process for reviewing user permissions, identifying excessive privileges, and removing unnecessary access.

Key Areas Covered:

• System Ownership Accountability
• User Access Validation
• Privileged Account Reviews
• Quarterly Access Certifications
• Separation of Duties Verification
• Access Recertification Procedures
• Joiner-Mover-Leaver Reviews
• Audit Evidence Collection

Frameworks Referenced:

• NIST SP 800-53 AC Family
• CIS Controls
• ISO 27001 Access Control Requirements

### MFA & Password Security Standard

Developed a Multi-Factor Authentication (MFA) and Password Security Standard to strengthen identity protection and reduce the risk of unauthorized access across IsoTech Ink's technology environment.

The standard establishes password creation requirements, authentication controls, password lifecycle management procedures, MFA enforcement requirements, and user responsibilities for protecting organizational accounts.

This document was created to address risks identified during the Initial Cybersecurity Risk Assessment, including weak password practices, remote workforce security concerns, credential theft, phishing exposure, and inconsistent authentication controls.

Key Areas Covered:

• Password Complexity Requirements
• Password Length Standards
• Multi-Factor Authentication (MFA)
• Password Storage & Protection
• Account Lockout Controls
• Credential Management
• Remote Access Authentication
• Privileged Account Security

Frameworks Referenced:

• NIST SP 800-63 Digital Identity Guidelines
• NIST SP 800-53
• CIS Critical Security Controls
• ISO 27001 Access Control Requirements
